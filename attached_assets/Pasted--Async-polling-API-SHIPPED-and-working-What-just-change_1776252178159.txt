Key changes for Replit engineer
Change the POST flow to not wait for the full pipeline. It returns 202 in ~100ms.
Implement polling against GET /api/mine/job/:jobId every 2-3 seconds.
Render whatever's in partialResults on each poll — don't wait for status === "done".
Stop polling when status === "done" or status === "error".
Client timeouts: POST = 5s, each GET = 5s. No more 180s timeouts.
Error state: check job.error — if non-null, the pipeline failed. Display job.error.message.
Job TTL: jobs auto-expire 30 min after completion. If user walks away and comes back to a stale jobId, they'll get 404. Just re-run the discovery.
For Replit's existing code that might still use sync mode
If they can't update to polling immediately, POST /api/mine?sync=true still works like the old behavior. Extend their client timeout to 180s and it'll keep working. But polling is strongly recommended for the UX win.
