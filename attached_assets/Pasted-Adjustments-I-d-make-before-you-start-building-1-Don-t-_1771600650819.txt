Adjustments I’d make before you start building
1) Don’t deduplicate “by domain” in the citation crawler

You wrote: “deduplicate by domain”. That will throw away useful pages (e.g., multiple Forbes pages; multiple G2 pages; competitor product pages vs blog pages).

Do:

Canonicalize URL (strip tracking params, resolve redirects)

Dedup by canonical URL

Then dedup by content hash (syndicated duplicates)
Keep domain as a feature, not a dedup key.

Why: domain-level dedup will create false “authority” gaps and starve the evidence collector.

2) Comparison targets should be selected per segment with a stable tie-breaker

Your selector is mostly right, but add one rule:

If two competitors are close, prefer:

higher presence in that segment

tie-breaker: higher retrieved authority within that segment

This makes the “vs A vs B” feel consistent and not random.

Also store:

segmentTopK (top 5 brands) in the report payload so you can later expand without reruns.

3) ContextConsistency needs a segment “intent dictionary” generator

Since you’re only doing category + audience (ignoring geo), make it deterministic:

For each segment, derive:

category_terms[]

audience_terms[]
from the prompt template / segment metadata (not from GPT).

Example:

Segment “Corporate cards for SMBs”

category terms: ["corporate card", "virtual card", "business card", "spend card"]

audience terms: ["SMB", "SME", "small business"]

Keep it a config map. This is MVP-safe and avoids LLM drift.

4) Snippet extraction: use variable window + structural capture

±25 tokens works for paragraphs but fails on:

listicles (bullet items)

tables (rows/cells)

headings

Do:

if mention is inside a list item → extract full list item

if inside a table row → extract row text

else fallback to ±25–40 tokens

This single change massively improves evidence quality.

5) ComparativePresence: don’t require multi-brand to be valuable

Your scoring already treats it as bonus, but your evidence collector must handle:

“comparison surfaces where you are absent”

“directories/listicles with only 1 tracked brand detected” (often because your competitor list is incomplete)

So for a comparison surface:

always extract top N entities even if they aren’t in your tracked set (basic NER-lite / capitalization heuristic is enough for MVP)

then show: “This page lists many providers; you are absent.”

Otherwise the feature often produces “0 brands found” which feels broken.

6) RetrievedAuthority should be computed from “unique domains with supporting snippets”

Not just “brand mentioned”.

Define “supporting snippet” as:

category match ≥ 1 OR audience match ≥ 1

Then authority counts “domains that reinforce the segment claim”, not random mentions.

This prevents garbage authority inflation.

7) Evidence collector: show 2 snippets per factor, not 1–3

Pick a consistent display rule:

2 snippets per factor per brand max (winner + you)

1 snippet if only one exists

Why: 3 becomes noisy; 1 feels cherry-picked.

8) Action recommender: keep it templated, not LLM-generated (MVP)

For MVP, make recommendations deterministic by gap type:

If category gap:

“Add explicit category phrase to: homepage H1 + first 200 words + G2/Crunchbase description”

If audience gap:

“Add explicit audience phrase: ‘for SMBs/startups’ in same sentence as category”

If comparative presence gap:

“Get listed on X comparison surface(s) where competitors appear; prioritize Tier1/2 domains in your retrieved set”

You can add LLM later for phrasing polish, but deterministic actions reduce hallucinations and support load.
