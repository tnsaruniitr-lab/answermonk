Good thing to think about. The current rate limit only covers per-IP, which is easy to bypass with rotating proxies. Here are the realistic layers of defence against that:

What already helps (partial protection):

Domain deduplication — if they submit the same domain from different IPs, the second request just returns the cached result. No new analysis runs. So rotating IPs on the same domain costs you nothing.
What's missing (gaps to fill):

CAPTCHA — the most effective single addition. Cloudflare Turnstile (free) sits in front of the submit button and blocks bots/automation without annoying real users with image puzzles. Rotating IPs doesn't help if you can't pass the CAPTCHA.

Global daily submission cap — regardless of IP, limit total new analyses to say 50/day. Since each analysis costs real money (AI API calls), this caps your worst-case exposure.

Email-gate before analysis — require an email address before the audit runs. Bots can fake emails but it adds friction and gives you a paper trail. You already have an email waitlist flow built in for the queue — this would just move it earlier.

Cloudflare proxy — once answermonk.ai is live and routed through Cloudflare, you get bot scoring, IP reputation, and DDoS protection for free on their free plan.

Recommended order of priority: Cloudflare proxy first (free, covers the most ground), then email-gate, then Turnstile CAPTCHA if you still see abuse.
