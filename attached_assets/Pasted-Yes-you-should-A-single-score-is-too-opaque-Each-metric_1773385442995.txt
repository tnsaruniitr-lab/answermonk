Best UI pattern

For every metric card, add a dropdown with 3 layers:

1. Sub-signals

Example for Trust:

DHA mentions

licensed mentions

certified mentions

JCI mentions

doctor-led mentions

trained professionals mentions

2. Counts

For each sub-signal show:

snippet numerator / denominator

domain numerator / denominator

own-site vs external split

Example:

DHA: 18/42 snippets, 6/10 domains

JCI: 2/42 snippets, 1/10 domains

3. Evidence examples

Show 3–5 matched snippets with:

source

URL/host

raw text highlight

That makes it visible and debuggable.

How to achieve this technically

Store metrics in a nested structure like:

{
  "trust": {
    "score": 54,
    "subsignals": {
      "dha": {"snippet_num": 18, "snippet_den": 42, "domain_num": 6, "domain_den": 10},
      "licensed": {...},
      "jci": {...}
    },
    "examples": [
      {"host": "example.com", "text": "DHA-licensed nurses..."},
      {"host": "directory.com", "text": "certified professionals..."}
    ]
  }
}

Then the UI can render:

top score

dropdown table

evidence snippets
