Must-fix now
A) Territory scoring: use weighted overlap, not raw counts

Raw counts bias toward territories with longer keyword lists and toward generic words like “creative”, “video”, “content”.

Change: give each keyword a weight (still deterministic):

strong phrases (multi-word): weight 3 (e.g., “paid search”, “marketing partner”, “visual identity”, “public relations”)

medium: weight 2 (e.g., “ppc”, “ecommerce”, “influencer”, “crm”, “reputation”)

generic: weight 1 (e.g., “content”, “video”, “creative”, “media”)

Score = sum(weights of matched keywords), not match count.
Still store matched keywords for explainability.

Why this matters: prevents “Social/Content” winning just because “content” appears 20 times.

B) Keyword matching: add phrase-first, then token match

If you just do token overlap, “paid” + “media” might match separately and inflate.

Rule:

match multi-word phrases on normalized text first (substring on tokenized text)

then match single tokens

don’t double-count overlaps for the same keyword
