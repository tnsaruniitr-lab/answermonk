Problem 1 — The How they appear line is being deliberately suppressed
Find this block in your TacticCard component:
javascriptconst bAppearClean = (
  bAppear === "NOT FOUND - Web search not available" ||
  bAppear.toLowerCase().includes("unattributed") ||
  bAppear.toLowerCase().includes("web verification") ||
  bAppear.toLowerCase().includes("web search not available") ||

This is stripping out the How they appear line whenever it contains any qualification language. So when Claude returns partial content like the run 18 output for tactic 1 — which had real quoted language mixed with a caveat — your UI throws the whole thing away. The user sees nothing instead of seeing something imperfect.
Problem 2 — The actions section is never rendered
Search your entire StructuredReport component. There is no rendering logic for actions or gap_analysis anywhere in it. The component renders summary, tactics, sources, unusual_findings, cross_engine_champions — but the actions object Claude returns is silently dropped. It is in the JSON but your UI never displays it.
Problem 3 — The new prompt format fields are not mapped
Your TacticCard normalises field names with this logic:
javascriptconst brandPerf: any[] = anyT.brand_performance ?? anyT.brands ?? anyT.brand_results ?? [];
But the new prompt returns brand_performance as the field name inside each tactic which should match. The issue is the evidence field — your code does:
javascriptconst rawEvidence: any[] = bp.evidence_urls ?? bp.evidence ?? [];
The new prompt returns evidence_urls as an array of objects with url and count fields. But run 18 returned them as strings like "https://www.sadu.vc — 33" which your parser tries to handle with a regex match. That regex is:

This only matches a single dash separator. The actual string uses — with em-dashes which the regex does not catch, so evidence URLs are silently dropped.

Fix 3 — Fix the em-dash evidence URL parser
Replace the regex in your evidence parsing with one that handles both dash types:
